## 常見調整怎麼做

### A. 修改 / 新增理解題
1. 編輯 `data/comprehension/{ID}.json`。格式：`{"章號": [{"q":"題目","options":["A","B","C","D"],"answer":"正確選項","basis":"經卷 章:節"}, ...]}`
2. **規則**：每題正好 4 個選項、答案字串必須完全等於某個選項、選項不重複、附 basis、題目與選項不可有 3 個以上連續英文字母（會被驗證擋）
3. **驗證腳本**（改完必跑，會抓答案不在選項/選項重複/缺依據/英文亂字）：
   ```
   cd C:/Users/User/LittleRainCC/bibleduo && node -e "const fs=require('fs');const id='MAL';const c=JSON.parse(fs.readFileSync('data/comprehension/'+id+'.json','utf8'));const b=JSON.parse(fs.readFileSync('data/books/'+id+'.json','utf8'));let e=[];if(Object.keys(c.questions).length!==b.chapters.length)e.push('章數不齊');for(const[ch,qs]of Object.entries(c.questions))for(const q of qs){if(!q.options.includes(q.answer))e.push(ch+':答案不在選項');if(q.options.length!==4||new Set(q.options).size!==4)e.push(ch+':選項問題');if(!q.basis)e.push(ch+':缺依據');if(/[a-zA-Z]{3,}/.test(q.q+q.options.join('')+q.answer))e.push(ch+':英文亂字');}console.log(e.length?e.join('; '):'OK');"
   ```
4. 提交推送（見下方部署流程）

## 部署流程（改完上線）

本機 `bibleduo/` 是獨立 git repo，直接 push 就會自動觸發 GitHub Pages 部署（約 1-2 分鐘生效）。**一律用 `git -C 絕對路徑` 明確指定倉庫**（母倉庫是 kindms/my-agent，不要推錯）：

```
git -C C:/Users/User/LittleRainCC/bibleduo add -A
git -C C:/Users/User/LittleRainCC/bibleduo commit -q -m "訊息

Co-Authored-By: Claude <noreply@anthropic.com>"
git -C C:/Users/User/LittleRainCC/bibleduo push -q
```

驗證正式站用 Playwright MCP（`browser_navigate` 到 https://kindms.github.io/bibleduo/ ，`browser_evaluate` 跑 fetch 檢查）。等部署可用 PowerShell 輪詢：`Invoke-WebRequest` 檢查 200。

## 已知的坑（踩過的雷）

1. **本機 curl 連不上 github.io / raw.githubusercontent**（SSL error 35/000）→ 下載或探測一律改用 **PowerShell `Invoke-WebRequest`**
2. **Chrome 擴充套件不能開 localhost 和 github.io** → 正式站驗證改用 **Playwright MCP**；Firebase 後台操作才用 claude-in-chrome
3. **preview_click 在手機模擬視窗座標會偏移（×1.1）** → 用 `preview_eval` 的 `.click()` 驗證
4. **理解題選項別放同義的正確答案**（例如「三年半」和「一千二百六十天」等價會變成兩個對的）
5. **CRLF 警告可忽略**（git 自動轉換 LF↔CRLF，不影響功能）
6. **App 只讀 PSA.json** 不讀分批暫存檔——詩篇若要再分批編輯，記得最後合併回 PSA.json
